Users of Agents

A one-page field guide for the inference-team PM · MECE by primary economic role w.r.t. the serving layer · 2026-06-15

The one insight. The literal user of an inference service is increasingly the agent — a machine client emitting a long, stateful, re-entrant request stream — not the humans/orgs using agents. NVIDIA inference sells directly to ≤3 segments; the rest are demand-shapers, SLO-setters, or standards levers. Figures directional unless cited; Dynamo 85–97% = realized single-session under its own routing.

Where to spend roadmap — the three High bets

1Operators

The only direct check-writer for the serving stack.

Build first: agent_hints + per-tenant KV / GPU-second attribution as a measured feature; prove the goodput delta on open infra.

2Model labs

Largest spend + biggest substitution risk (ASICs).

Don’t fight kernels — win the seam: cross-tenant KV-locality control plane + day-one MoE serving. Cost-per-verified-task as the trojan horse.

3Enterprise

Biggest net-new durable TAM; can’t churn to an API.

Package: a “governed agent factory in a box” — NIM + on-prem serving + cost/ROI attribution as one SKU.

The MECE rule that keeps it clean

  • Role precedence: lab > operator > builder. A firm that trains the weights is scored as a lab even if it also runs GPUs and ships agents (OpenAI’s fleet, Claude Code).
  • GPU custody = the operator test. Own the fleet & accountable for $/token = operator; otherwise it’s tooling (routers, gateways).
  • Builder vs. enterprise = AI-native & sold to third parties. Cursor/Harvey = builder; Klarna’s support agent / Agentforce = enterprise.
  • Demote the workload. “Agent-as-user” is the request-pattern spec, not a peer actor — else you double-count every request.

So what for the PM

  • One buyer signs the check (operators); one sets the spec (the agent); one is the durable land-grab (enterprise); one is the strategic seam (labs).
  • The wedge is the same everywhere: per-tenant, per-task cost + KV-locality attribution + the hint / continuity contract.
  • GRIT’s measurement (cost-per-verified-task, the locality tax) is the neutral evidence that makes that wedge legible to all of them.
